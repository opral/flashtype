# Flashtype ICP

## Definition

People using Claude, Codex, Claude Code, ChatGPT, or another AI agent with local Markdown files, docs, notes, wikis, or project workspaces.

Job title is secondary. The primary signal is behavior:

- They use Markdown or local files.
- They use Claude/Codex/AI agents on those files.
- The files are revised repeatedly.
- The agent workflow is awkward because the editor/workspace and the agent do not meet cleanly.

## Why Now

Claude and Codex want local files, and they work especially well with Markdown. That creates a new segment of knowledge workers who are moving serious writing into local Markdown because AI agents can edit it directly.

This is not just a developer-docs workflow anymore. It is a new local-file knowledge-work pattern.

## Core PULL

Active work:

- Write, revise, and maintain durable docs, notes, wikis, and Markdown workspaces with AI.

Situation:

- Claude/Codex can now edit local files directly.
- Markdown is the easiest document format for these agents to read and change.
- People adopting this workflow need the agent to work inside a durable, human-friendly workspace.

Current options:

- Claude/Codex/ChatGPT in chat or terminal.
- Obsidian or another Markdown editor.
- Manual copies of files.
- Git, GitHub Desktop, VS Code, or Cursor for technical users.
- Google Docs / Notion when the workflow is not local-file based.

Limitations:

- Chat copy-paste makes changes hard to inspect.
- Obsidian and Markdown editors often require external Claude/Codex wiring.
- Git is often too developer-shaped for knowledge workers.
- VS Code/Cursor/Git may be good enough for technical users, so dev campaigns need a stronger WYSIWYG writing angle.
- Google Docs and Notion are comfortable editors, but they do not fit Claude/Codex local-file editing as directly.

## Strong Signals

- Posts about using Claude/Codex/Claude Code with Markdown, local files, docs, notes, specs, or README files.
- Comments asking how to connect Claude/Codex to a writing workspace, vault, folder, or wiki.
- Obsidian users discussing Claude/Codex/AI editing vaults.
- People trying to connect AI agents to a folder, vault, repo, or local writing workspace.
- Posts about `CLAUDE.md`, `AGENTS.md`, agent instructions, AI-written docs, or local knowledge-work files.
- Complaints about copy-paste, external agent setup, brittle Obsidian wiring, Notion export/API/MCP friction, Git setup, or split sources of truth.

## Good Fit

- The person is visibly using local Markdown and AI together.
- The work recurs, not a one-off note.
- The document matters enough that review/control matters.
- They are early enough in the workflow that the current toolchain is awkward.

## Weak Fit

- The only signal is job title.
- The only signal is "writes docs" without Claude/Codex/local-file evidence.
- The person is a developer already happy in VS Code/Cursor/Git.
- The workflow is purely Google Docs/Notion/chat with no pull toward local files.
